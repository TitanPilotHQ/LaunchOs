# Titan Pilot Design System v1

## Purpose

Define the visual and interaction language for Titan Pilot across website, product demos, investor assets, launch films, and social content.

## Design Principles

1. Calm authority
2. Technical clarity
3. Premium restraint
4. Financial seriousness
5. AI without gimmicks
6. Motion with purpose
7. Trust before spectacle

## Typography

### Recommended Stack

- Headings: Inter, Geist, or Satoshi
- Body: Inter or Geist Sans
- Monospace: JetBrains Mono or Geist Mono

### Typographic Tone

Large headings should feel cinematic and precise. Body copy should be clear, calm, and technical without becoming dense.

## Layout System

- 12-column desktop grid
- 8-column tablet grid
- 4-column mobile grid
- generous spacing
- dark negative space
- narrow text measure for long-form reading

## Component Language

### Cards

Cards should look like mission-control panels, not generic SaaS tiles.

Properties:
- subtle border
- deep translucent surface
- soft inner glow
- clear heading
- concise copy
- optional metric or status badge

### Buttons

Primary button:
- dark surface
- cyan/blue gradient border
- subtle glow on hover

Secondary button:
- transparent surface
- muted border
- high contrast text

### Badges

Use badges for:
- Replay Verified
- Risk Gate
- AI Shadow
- Broker Connected
- Fail Closed
- Audit Ready

## Motion System

Motion should communicate system intelligence.

Allowed:
- slow reveal
- connection line drawing
- data stream movement
- glow pulse
- chart trace animation
- card depth hover

Avoid:
- bouncing UI
- noisy particles
- crypto-style flashing
- fake urgency

## Chart Language

Charts should be clean, minimal, and evidence-driven. Avoid exaggerated profit curves.

## Iconography

Use line icons with consistent stroke width. Icon categories:

- AI reasoning
- deterministic control
- replay
- broker truth
- risk gate
- verification
- recovery
- infrastructure

## Imagery

Prefer abstract financial infrastructure visuals over lifestyle imagery.

Recommended themes:
- dark data centers
- holographic market graphs
- neural topology
- glass dashboards
- system diagrams

## Accessibility

- high contrast text
- visible focus states
- reduced motion support
- no core meaning conveyed by color alone

## Design Rule

If a visual element does not increase trust, clarity, or product understanding, remove it.
